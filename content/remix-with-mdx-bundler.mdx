## 使用付費服務的原因 & 費用

### [fly.io](https://fly.io)

想要使用 `MDX`，但是 Remix 目前把 `.mdx` 檔案當做是頁面的功能還非常陽春，例如：無法取得路徑名稱，這樣連基本的 SEO 設定都無法滿足。因此，以 Remix 做的網站（部落格），有許多是 `mdx-bundler`，這也是 Remix 開發者做的套件。

一開始使用 [Vercel](https://vercel.com)，發現 `mdx-bundler` 要用到的 `fs` 在開發環境可以使用，但是推上 `Vercel` build 成功，打開預覽網址才遇到錯誤訊息：

```txt
There was an error
ENOENT: no such file or directory, open '/var/task/content/postname.mdx'
```

Discord 上面也有人嘗試過：[MDX blog in Remix.run](https://blog.oldweb2.com/remix-mdx-blog)，但最後結論就是改用 [fly.io](https://fly.io)。

### 調整記憶體

Remix 在安裝的時候就有提供 fly.io 的設定，要能夠跑 Remix， Memory 必須調整為 512MB 才跑得動。

![在 fly.io 設定 Memory 為 512MB](/images/remix-with-mdx-bundler/flyIoMemory.webp)

### 費用

2022 年 2 月使用一個月的費用是：0.95 鎂。

## Markdown 外部連結

Markdown 的超連結語法：

```markdown
[Remix 首頁](https://remix.run)
```

會產生基本的超連結：
```html
<a href="https://remix.run">Remix 首頁</a>
```

但是，想要以新開分頁的方式開啟，而現在外部連結都強調安全性，應該要：
```html
<a href="https://remix.run" target="_blank" rel="noopener">Remix 首頁</a>
```

`mdx-bundler` 的 [Component Substitution](https://github.com/kentcdodds/mdx-bundler#component-substitution) 可以處理這個願望：

判斷「外部連結」都是以 `http` 開頭，就回傳有 `target` 和 `rel` 的 `<AnchorLink>` 元件。

```jsx
import { getMDXComponent, getMDXExport } from 'mdx-bundler/client';
import { useMemo } from 'react';

function TagForAnchor({ children, href }) {
  if (!href.startsWith('http')) {
    return <AnchorLink href={href}>{...children}</AnchorLink>
  }

  return <AnchorLink href={href} target="_blank" rel="noopener">{children}</AnchorLink>
}

export default function Post() {
  const { code } = useLoaderData<LoaderData>();
  const Component = useMemo(() => getMDXComponent(code), [code]);
  return (
    <Component components={{
      a: TagForAnchor
    }} />
  );
}
```
